# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

#### Данные
Интерфейс для товара:
interface IProduct {
  id - уникальный идентификатор товара.
  title - название товара.
  description - описание товара.
  image - ссылка на изображение товара.
  category - категория, к которой относится товар.
  price - цена товара.
}

Интерфейс для покупателя:
interface IBuyer {
  payment - выбранный способ оплаты.
  email - email покупателя.
  phone - телефон покупателя.
  address - адрес доставки.
}

Интерфейс информации о товаре и покупателе для отправки на сервер для POST запроса:
interface Order extends IBuyer { - остальное унаследованно от IBuyer.
  items - массив товаров в заказе.
  total - общая стоимость заказа.
}

Интерфейс получения информации от сервера для GET запроса:
interface Product {
  items - массив товаров полученных от сервера.
  total - количество товаров.
}

Интерфейс ответа сервера после успешного POST запроса:
interface OrderApi {
  id - идентификатор покупателя хранящийся на сервере.
  total - общая стоимость покупок.
}

#### Модели данных
##### class Buyer
Класс содержащий информацию о введенных данных покупателя.

Конструктор класса не принимает параметров.

Поля класса:
`private payment: IBuyer["payment"] = ""` - выбранный способ оплаты (приватный).
`private email: string = ""` - введенная электронная почта (приватный).
`private phone: string = ""` - введенный телефон (приватный).
`private address: string = ""` - введенный адрес (приватный).

Методы класса:
`setModel(buyer: Partial<IBuyer>)` - добавляет новые данные покупателя. Принимает объект данных пользователя.
`getBuyer(): IBuyer` - получение данных о покупателя.
`clearBuyer(): void` - очистка данных о покупателе.
`validate(): ValidError` - проверка на незаполненные поля.

##### class Cart
Класс хранящий массив выбранных товаров.

Конструктор класса не принимает параметров.

Поля класса:
`private items: IProduct[] = []` - хранит массив товаров корзины (приватный).

Методы класса:
`getItems(): IProduct[]` - получение хранящегося массива выбранных товаров.
`addItem(item: IProduct): void` - добавление данных в корзину. Принимает объект с данными товара в качестве параметра.
`clearItem(item: IProduct)` - удаление товара по его id из корзины. Принимает объект с данными товара в качестве параметра.
`clearCart(): void` - удаление всех товаров из корзины.
`sumItems(): number` - получение стоимости всех товаров из корзины.
`maxItems(): number` - получение количества товаров в корзине.
`searchItem(id: string)` - проверка наличия товара по id в корзине. Принимает id товара в качестве параметра.

##### class Catalog
Класс который хранит как массив всех товаров, так и выбранных товаров.

Конструктор класса не принимает параметров.

Поля класса:
`private products: IProduct[] = []` - массив все товаров (приватный).
`private thisProduct: IProduct | null = null` - выбранный товар (приватный).

Методы класса:
`setItems(products: IProduct[]): void` - сохранение массива все товаров. Принимает массив товаров в качестве параметра.
`getItems(): IProduct[]` - получение хранящегося массива товаров
`getProductsId(id: string): IProduct | undefined` - получение товара по его id. Принимает id товара в качестве параметра.
`setThisProduct(Product: IProduct): void` - сохранение выбранного товара. Принимает объект товара в качестве параметра.
`getThisProduct(): IProduct | null` - получение хранящегося выбранного товара.

#### Слой представления
##### class Header
Класс, представляющий компонент шапки сайта с кнопкой корзины и счетчиком.

Интерфейс `IHeader` определяет структуру данных:
`counter: number` — число, отображающее количество элементов.

Конструктор класса принимает контейнер DOM-элемента и объект событий `events`. Инициализируются элементы `counterElement` и `basketButton` с помощью функции `ensureElement`.  
На кнопку `basketButton` навешивается обработчик клика, который вызывает событие `basket:open` через объект `events`.

Поля класса:
`protected counterElement: HTMLElement` — элемент для отображения количества товаров в корзине.`protected basketButton: HTMLButtonElement` — кнопка для открытия корзины.

Методы класса:
`set counter(value: number): void` — обновляет текстовое содержимое счетчика количеством товаров.

##### class Gallery
Класс, управляющий отображением галереи с элементами каталога.

Использует интерфейс `IGallery`, описывающий структуру объекта галереи с массивом элементов каталога.
Поля интерфейса:
`catalog: HTMLElement[]` — массив элементов DOM, представляющих элементы галереи.

Конструктор класса принимает контейнер DOM-элемента и объект событий `event`. Инициализируется `catalogElement` с помощью функции `ensureElement`, ищущей элемент с классом `.gallery` внутри контейнера.

Поля класса:
`protected catalogElement: HTMLElement` — элемент DOM, в который вставляются элементы галереи.

Методы класса:
`set catalog(items: HTMLElement[]): void` — обновляет содержимое галереи, заменяя текущие элементы на новые, переданные в массиве items.

##### class Basket
Класс, управляющий отображением корзины с товарным списком и ценой.

Использует интерфейс `IBasket`, описывающий структуру объекта корзины с массивом элементов товаров и общей ценой.
Поля интерфейса:
`list: HTMLElement[]` — массив элементов DOM, представляющих товары в корзине.
`price: number` — общая стоимость товаров в корзине.

Конструктор класса принимает контейнер DOM-элемента и объект событий events. Инициализируется basketList, basketPrice и basketButton с помощью функции ensureElement, ищущей соответствующие элементы внутри контейнера:
`.basket__list`
`.basket__price`
`.basket__button`
В конструкторе также навешивается обработчик события `click` на кнопку `basketButton`, который вызывает событие `"basket:submit"` через объект `events`. Изначально `basketButton` отключена `(disabled = true)`.

Поля класса:
`protected basketList: HTMLElement` — элемент DOM, в который вставляются элементы товаров корзины.
`protected basketPrice: HTMLElement` — элемент DOM, отображающий общую цену.
`protected basketButton: HTMLButtonElement` — кнопка для подтверждения или оформления заказа.

Методы класса:
`set list(items: HTMLElement[]): void` — обновляет содержимое списка товаров, заменяя текущие элементы на новые. Если список пустой, кнопка basketButton отключается, иначе — включается.
`set price(value: number): void` — обновляет отображение цены, добавляя к значению текст "синапсов".

##### class Modal
Класс, управляющий отображением модального окна с содержимым.

Использует интерфейс `IModal`, описывающий структуру объекта модального окна с содержимым.
Поля интерфейса:
`content: HTMLElement` — элемент DOM, который отображается внутри модального окна.

Конструктор класса принимает контейнер DOM-элемента и объект событий `events`. Инициализируется `modalContent` и `modalClose` с помощью функции `ensureElement`, ищущей элементы с классами .`modal__content` и `.modal__close` внутри контейнера.
В конструкторе навешиваются обработчики:
На кнопку `modalClose`, которая вызывает событие `"modal:close"` при клике.
На сам контейнер, который при клике по области вне содержимого также вызывает событие `"modal:close"`.

Поля класса:
`protected modalContent: HTMLElement` — элемент DOM, в который вставляется содержимое модального окна.
`protected modalClose: HTMLButtonElement` — кнопка закрытия модального окна.

Методы класса:
`set content(item: HTMLElement)` — заменяет содержимое модального окна на переданный элемент и открывает окно.
`open(): void` — добавляет класс "modal_active" к контейнеру для отображения окна.
`close(): void` — удаляет класс "modal_active" для скрытия окна.

##### class Success
Класс, управляющий отображением сообщения об успешной операции с информацией о списанных синапсах.

Использует интерфейс `ISuccess`, описывающий структуру объекта успеха с полем `total`.
Поля интерфейса:
`total: number` — количество списанных синапсов.

Конструктор класса принимает контейнер DOM-элемента и объект событий `events`. Инициализируется `orderSuccess` и `buttonSuccess` с помощью функции `ensureElement`, ищущих элементы с классами `.order-success__description` и `.order-success__close` внутри контейнера.
В конструкторе навешивается обработчик события `click` на кнопку `buttonSuccess`, который вызывает событие `"success:close"` через объект `events`.

Поля класса:
`protected orderSuccess: HTMLElement` — элемент DOM, отображающий сообщение об успехе.
`protected buttonSuccess: HTMLButtonElement` — кнопка закрытия сообщения.

Методы класса:
`set total(value: number)` — обновляет текстовое содержимое элемента orderSuccess, показывая, сколько синапсов списано, в формате: Списано ${value} синапсов.

`protected cardTitle: HTMLElement` — элемент DOM для отображения заголовка.
`protected cardPrice: HTMLElement` — элемент DOM для отображения цены.

Методы класса:
`set title(value: string)` — устанавливает текст заголовка.
`set price(value: number | null)` — обновляет отображение цены: если value не null, показывает ${value} синапсов, иначе — 'Бесценно'.

##### class CardBasket

Поля класса:
`protected itemId: HTMLElement` — элемент DOM, отображающий идентификатор элемента корзины.
`protected buttonClose: HTMLButtonElement` — кнопка для закрытия или удаления элемента.

Методы класса:
`set id(value: number)` — обновляет текстовое содержимое элемента itemId, показывая значение value в виде строки.

`button: string` — текст кнопки.
`text: string` — описание текста.

Конструктор класса принимает контейнер DOM-элемента и объект событий `event`. Инициализируются `cardCategory`, `cardImage`, `cardButton` и `cardText` с помощью функции `ensureElement`, ищущих соответствующие элементы внутри контейнера.
Навешивается обработчик события `click` на кнопку `cardButton`, который вызывает метод `emit` у объекта `event` с событием `"product:choose"`.

Поля класса:
`protected cardCategory: HTMLElement` — элемент DOM, отображающий категорию.
`protected cardImage: HTMLImageElement` — изображение товара.
`protected cardButton: HTMLButtonElement` — кнопка для выбора продукта.
`protected cardText: HTMLElement` — элемент для отображения описательного текста.

Методы класса:
`set category(value: string)` — обновляет текст и класс элемента cardCategory, находя соответствующую категорию в объекте categoryMap и добавляя соответствующий класс.
`set image(src: string)` — устанавливает источник изображения, добавляя базовый URL CDN_URL.
`set text(value: string)` — обновляет содержимое текста описания.
`set button(value: string)` — обновляет текст на кнопке.

##### class Form
Класс, управляющий формой с возможностью отображения ошибок и кнопкой отправки.

Использует интерфейс `IForm`, который содержит:
`error: string` — сообщение об ошибке.

Класс `Form<T>` расширяет базовый класс `Component`, принимая универсальный тип `T` и добавляя к нему свойства `IForm`. Инициализируются `formButton` и `formError` с помощью функции `ensureElement`, ищущих элементы по селекторам внутри контейнера:
`".button[type=submit]"` для кнопки отправки.
`".form__errors"` для отображения ошибок. Навешивается событие `click` на кнопку `formButton`, которое вызывает `preventDefault()`, чтобы предотвратить стандартное поведение отправки формы.

Поля класса:
`protected formButton: HTMLButtonElement` — кнопка отправки формы.
`protected formError: HTMLElement` — элемент для отображения сообщения об ошибке.

Метод:
`set error(value: string)` — обновляет текст ошибки, отображая сообщение в элементе formError.

##### class ContactForm
Класс, реализующий контактную форму с полями для ввода email и телефона, а также с логикой отправки и валидации.

Использует интерфейс `IContactForm`, который включает:
`email: string` — адрес электронной почты.
`phone: string` — номер телефона.

Класс `ContactForm` расширяет `Form<IContactForm>`. Инициализирует поля `inputEmail` и `inputPhone` с помощью функции `ensureElement`, ищущих элементы по селекторам внутри контейнера:
`.form__input[name=email]` для email.
`.form__input[name=phone]` для телефона.
Навешивает обработчики `input` на оба поля:
При вводе в email генерируется событие `input:email` с текущим значением.
При вводе в телефон — событие `input:phone`. Навешивает событие `click` на кнопку формы, которое вызывает событие `contact:submit` через объект `event`.

Поля класса:
`protected inputEmail: HTMLInputElement` — поле для ввода email.
`protected inputPhone: HTMLInputElement` — поле для ввода номера телефона.

Методы:
`set email(value: string)` — обновляет значение email и вызывает validateContact().
`set phone(value: string)` — обновляет значение телефона и вызывает validateContact().
`validateContact()` проверяет, что оба поля не пустые:
Если заполнены оба — удаляет атрибут disabled у кнопки отправки.
Если одно из них пустое — устанавливает атрибут disabled, делая кнопку неактивной.

##### class OrderForm
Класс, реализующий форму заказа с выбором способа оплаты и адресом доставки.

Использует интерфейс `IOrderForm`, который содержит:
`payment: "card" | "cash"` — выбранный способ оплаты.
`address: string` — адрес доставки.

Класс `OrderForm` расширяет `Form<IOrderForm>` и внутри конструктора:
Инициализирует кнопки для выбора способа оплаты `(buttonCard и buttonCash)` через `ensureElement`.
Инициализирует поле для ввода адреса `(inputAddress)` через `ensureElement`.
Навешивает события:
`click на кнопку buttonCard, вызывающее событие buyer:card.`
`click на кнопку buttonCash, вызывающее событие buyer:cash.`
`input на поле inputAddress, вызывающее событие input:address с текущим значением.`
`click на кнопку формы, вызывающее событие order:submit.`

Поля класса:
`protected buttonCard: HTMLButtonElement` — кнопка выбора оплаты картой.
`protected buttonCash: HTMLButtonElement` — кнопка выбора оплаты наличными.
`protected inputAddress: HTMLInputElement` — поле для ввода адреса.

Методы:
`set payment(value: "card" | "cash")` — устанавливает выбранный способ оплаты, добавляя или удаляя класс button_alt-active у соответствующих кнопок, и вызывает validateContact().
`set address(value: string)` — обновляет значение адреса и вызывает validateContact().
`validateContact()` проверяет:
Есть ли адрес (hasAddress).
Выбран ли способ оплаты (paymentSelected).
Если оба условия выполнены — активирует кнопку formButton.
Иначе — блокирует кнопку, устанавливая атрибут disabled.

#### Презентер
Поскольку у приложения одна страница, всю логику обработки событий и взаимодействия компонентов можно реализовать в одном "презентере" или в основном управляющем скрипте. В этом случае, событие, например, `product:select`, слушается глобально, и в ответ вызывается метод модели `setSelectedProduct()`, который обновляет состояние выбранного товара. После этого модель эмитит событие `catalog:setSelectedProduct`. На него слушатель вызывает отображение модального окна с информацией о выбранном товаре через компонент представления.
При взаимодействии пользователя с кнопками добавления или удаления товаров из корзины, срабатывает событие `product:choose`. В зависимости от того, есть ли товар в корзине, вызывается либо `addProductsToBuy()`, либо `deleteProductsFromBuy()`. Метод `addProductsToBuy()` обновляет состояние корзины, эмитит событие `basket:change`, на которое слушатель реагирует, обновляя отображение товаров в корзине, нумерацию, итоговую сумму и счетчик в шапке страницы.
Также при изменении данных покупателя (например, выбора способа оплаты, ввода адреса или контактов), соответствующие события вызывают методы модели `setModel()`, которая валидирует текущие данные и эмитит события для обновления формы и отображения ошибок или подтверждений.
Когда пользователь подтверждает заказ, срабатывает событие `contact:submit`. В обработчике собираются все данные заказа, отправляются на API, и в случае успеха вызывается событие `api:post`, после чего происходит очистка корзины, сброс данных покупателя и отображение окна успеха.

Общий цикл включает:
прослушивание событий пользовательских действий,
вызов методов модели для обновления данных,
обновление интерфейса через компоненты представления,
взаимодействие с API для отправки заказа и получения данных каталога.
